# ElRepo: Tu Asistente para Crear Sitios Web

## Guía Completa para Principiantes

---

## 1. ¿Qué es ElRepo?

**ElRepo** es una **plantilla inteligente** que te ayuda a crear sitios y aplicaciones web con la ayuda de inteligencia artificial.

Imagina que quieres construir una casa. Podrías empezar desde cero: comprar los materiales, cortar la madera, mezclar el cemento... pero eso tomaría muchísimo tiempo y necesitarías saber mucho de construcción.

En cambio, ¿qué tal si tuvieras un **kit de construcción** que ya incluye:
- Los planos de diferentes tipos de casas
- Las herramientas necesarias
- Un asistente experto que te guía paso a paso

**ElRepo es ese "kit de construcción" pero para sitios web.**

### En términos sencillos:

ElRepo es una **carpeta especial** en tu computadora que contiene:
- **Plantillas pre-hechas** para diferentes tipos de proyectos web
- **Reglas y guías** para que todo quede bien hecho
- **Herramientas automáticas** que revisan tu trabajo
- **Instrucciones** que una inteligencia artificial puede seguir para ayudarte

## 3. ¿Qué funciones y ventajas ofrece ElRepo?

### 🎯 12 Tipos de Proyectos Pre-Configurados

ElRepo incluye **12 plantillas diferentes** ("starters") que puedes usar inmediatamente. Cada una está diseñada para un propósito específico.

### 🧠 54 "Habilidades" Inteligentes

Las **habilidades** son como "expertos virtuales" que se activan automáticamente según lo que estés haciendo:

| Habilidad | ¿Qué hace? |
|-----------|-----------|
| `xhtml-author` | Escribe el código HTML correctamente |
| `css-author` | Crea los estilos y colores del sitio |
| `forms` | Diseña formularios que funcionan bien |
| `accessibility-checker` | Verifica que todos puedan usar tu sitio |
| `responsive-images` | Ajusta las imágenes para diferentes dispositivos |
| `metadata` | Agrega información para que Google encuentre tu sitio |
| `performance` | Hace que tu sitio cargue rápido |
| `security` | Protege tu sitio contra problemas de seguridad |

### 🛠️ Más de 50 Comandos Especiales

Los **comandos** son instrucciones que le das a ElRepo para que haga cosas por ti. Funcionan como "atajos" que activan procesos automáticos.

**Comandos principales:**

| Comando | ¿Qué hace? |
|---------|-----------|
| `/init` | Inicia un proyecto nuevo desde cero |
| `/scaffold-site` | Crea un sitio web completo |
| `/scaffold-blog` | Crea un blog listo para usar |
| `/scaffold-ecommerce` | Crea una tienda en línea |
| `/add-pattern` | Agrega un diseño de página pre-hecho |
| `/add-theme` | Crea un tema de colores personalizado |
| `/fake-content` | Genera texto de ejemplo para tus páginas |
| `/placeholder-image` | Crea imágenes de prueba |
| `/health` | Revisa la salud general de tu proyecto |

### ✅ Herramientas de Revisión Automática

ElRepo incluye herramientas que **revisan tu trabajo automáticamente**:

| Herramienta | ¿Qué revisa? |
|------------|-------------|
| `npm run lint` | Que el código HTML esté bien escrito |
| `npm run lint:css` | Que los estilos no tengan errores |
| `npm run a11y` | Que tu sitio sea accesible para todos |
| `npm run lint:seo` | Que tu sitio sea fácil de encontrar en Google |
| `npm run lint:images` | Que las imágenes estén optimizadas |
| `npm test` | Que todo funcione correctamente |

### 🎨 Sistema de Diseño Incluido

ElRepo trae un **sistema de diseño completo**:

- **Colores pre-definidos** que combinan bien
- **Tipos de letra** que se leen fácilmente
- **Componentes reutilizables** (botones, tarjetas, menús)
- **Modo claro y oscuro** automático
- **Diseño adaptable** a celulares y tablets

### 📚 Biblioteca de Patrones

Los **patrones** son "piezas de LEGO" que puedes combinar para crear páginas:

| Categoría | Patrones incluidos |
|-----------|-------------------|
| **Navegación** | Menús, encabezados, migas de pan |

| **Feedback** | Alertas, mensajes, ventanas emergentes |

### 📋 Seguimiento de Tareas

ElRepo incluye un sistema para **llevar control de lo que haces**:


## 5. ¿Cómo se usa ElRepo en la práctica?

### Paso 1: Preparación

Antes de empezar, necesitas tener instalado:

1. **Node.js** (versión 18 o superior) - Es un programa gratuito que permite ejecutar herramientas web
2. **npm** - Viene incluido con Node.js, es como una "tienda de aplicaciones"
3. **Un asistente de IA** - Como Claude Code, que entiende las instrucciones de ElRepo

### Paso 2: Clonar el Repositorio

"Clonar" significa **copiar ElRepo a tu computadora**:

```bash
git clone <dirección-del-repositorio> mi-proyecto
cd mi-proyecto
npm install
```

Esto hace dos cosas:
1. Copia todos los archivos de ElRepo a una carpeta en tu computadora
2. Descarga e instala las herramientas necesarias

### Paso 3: Iniciar tu Proyecto

Escribe el comando mágico:

```bash
/init
```

Este comando inicia un **asistente interactivo** que te hará preguntas como:

```
¡Bienvenido a Project Template!

¿Qué tipo de proyecto quieres crear?
○ Sitio Web Estático
○ Astro Site
○ PWA/SPA
○ REST API
○ Dashboard
○ Chrome Extension
○ CLI Tool
○ Sitio de Documentación
○ Blog
○ Form Builder
○ Design System
○ E-commerce Storefront
```

### Paso 4: Responder las Preguntas

El asistente te pedirá información:

```
Nombre del proyecto (en minúsculas, con guiones):
→ mi-tienda-en-linea

Descripción (máximo 160 caracteres):
→ Tienda de productos artesanales hechos a mano

Nombre del sitio (nombre visible):
→ Mi Tienda Artesanal

Color del tema (por defecto: #1e40af):
→ #1e40af
```

### Paso 5: ElRepo Crea Todo por Ti

Una vez que respondes, ElRepo:

1. **Crea la estructura de carpetas**
2. **Genera todos los archivos necesarios**
3. **Configura las herramientas**
4. **Te muestra qué hacer después**

### Paso 6: Trabajar en tu Proyecto

Ahora puedes empezar a trabajar. ElRepo te ayuda con:

```bash
npm run dev      # Inicia un servidor para ver tu sitio mientras lo creas
npm run lint     # Revisa que todo esté bien escrito
npm run test     # Ejecuta pruebas para asegurar que funciona
npm run build    # Prepara tu sitio para publicarlo en internet
```

## 6. ¿Cómo darle órdenes a ElRepo?

ElRepo se usa principalmente a través de **comandos** que escribes. Aquí te explico cómo funciona:

### Tipos de Comandos

#### A) Comandos de Inicialización (`/`)

Estos comandos **crean cosas nuevas**:

| Comando | Resultado |
|---------|-----------|
| `/init` | Inicia el asistente de creación de proyectos |
| `/scaffold-site` | Crea un sitio web completo |
| `/scaffold-blog` | Crea un blog |
| `/scaffold-ecommerce` | Crea una tienda en línea |
| `/add-pattern` | Agrega un diseño de página |
| `/add-theme` | Crea un tema de colores |
| `/add-element` | Define un nuevo componente HTML |
| `/add-form-field` | Agrega un campo de formulario |

**Ejemplo de uso:**

```
Tú: /scaffold-blog

ElRepo: ¡Claro! Voy a crear un blog para ti.

Primero, necesito saber:
1. ¿Cuál es el nombre del blog?
2. ¿Qué tipo de contenido publicarás?
3. ¿Quieres incluir comentarios?
```

#### B) Comandos de Revisión (`npm run`)

Estos comandos **revisan tu trabajo**:

| Comando | ¿Qué hace? |
|---------|-----------|
| `npm run lint` | Revisa el código HTML |
| `npm run lint:css` | Revisa los estilos |
| `npm run lint:js` | Revisa el JavaScript |
| `npm run a11y` | Verifica accesibilidad |
| `npm run lint:seo` | Analiza optimización para buscadores |
| `npm test` | Ejecuta pruebas automáticas |

**Ejemplo de uso:**

```
Tú: npm run lint

ElRepo: Revisando tu código...

✓ 15 archivos HTML revisados
✓ No se encontraron errores
✓ Todo está bien escrito
```

#### C) Comandos de Utilidad

Estos comandos **hacen tareas específicas**:

| Comando | ¿Qué hace? |
|---------|-----------|
| `/fake-content` | Genera texto de ejemplo |
| `/placeholder-image` | Crea imágenes de prueba |
| `/health` | Muestra el estado de tu proyecto |
| `/list-themes` | Muestra los temas de colores disponibles |
| `/switch-theme` | Cambia el tema de colores |

### Cómo Formular Instrucciones Correctamente

#### ✅ Forma CORRECTA:

**Sé específico sobre lo que quieres:**

```
"Quiero crear una página de contacto con:
- Un formulario con nombre, email y mensaje
- Un mapa de ubicación
- Información de contacto (teléfono, dirección)"
```

**Usa los comandos adecuados:**

```
/init              # Para empezar un proyecto nuevo
/scaffold-site     # Para crear un sitio web
/add-form-field    # Para agregar campos a un formulario
```

**Proporciona contexto:**

```
"Estoy creando un sitio para una panadería.
Necesito una página que muestre nuestros productos.
Los colores deben ser cálidos (naranja, marrón)."
```

#### ❌ Forma INCORRECTA:

**Demasiado vago:**

```
"Haz algo bonito"
```

**Sin usar los comandos:**

```
"¿Cómo creo un blog?"  (en lugar de usar /scaffold-blog)
```

**Sin dar contexto:**

```
"No funciona"  (sin explicar qué ni cómo)
```
